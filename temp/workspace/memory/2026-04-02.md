## Ollama Latency Investigation

**Trigger:** User X reported 30-second response delays in OpenClaw.

### Findings

1. **OpenRouter Free Model** (qwen/qwen3.6-plus-preview:free)
   - Free tier queue causes 30+ second delays during peak hours
   - HTTP endpoint latency ~1s (not counting queue wait)
   - User confirmed to switch to local model

2. **Ollama Local Model** (qwen3.5:9b on GPU)
   - Cold start: ~13.5s total (4.2s model load + 9s generation for 108 tokens)
   - Hot start: ~10.7s total (0.2s load + 10.25s generation for 123 tokens)
   - Generation speed: ~12 tokens/second
   - GPU: RTX 3060 12GB, model uses 5.73GB VRAM
   - Quantization: Q4_K_M

3. **Root Cause of Slower-Than-Expected Speed**
   - Forza Horizon 5 (FH5) was running concurrently
   - Game consumed ~11.7GB of 12GB VRAM, leaving only 424MB free
   - Ollama forced to share/swap, reducing speed from expected 30-40 t/s to ~12 t/s
   - Recommendation: close games before running Ollama, or use cloud model instead

### User Preferences Updated
- Prefers local Ollama model over OpenRouter free tier
- Values response speed investigation and optimization
