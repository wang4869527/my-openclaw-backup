# 2026-03-28 - OpenClaw 备份系统搭建

## 目标
配置 OpenClaw 自动备份到私有 GitHub 仓库，实现配置修改的安全回滚能力。

## 过程

### SSH 配置
- 生成 Ed25519 密钥对（~\.ssh\id_ed25519）
- 公钥添加到 GitHub 账户
- 测试 `ssh -T git@github.com` 成功

### 仓库初始化
- 创建 `C:\Users\Administrator\openclaw-backups`
- `git init`，添加远程仓库 `git@github.com:wang4869527/my-openclaw-backup.git`
- 首次尝试全量备份失败（179MB > 100MB 限制）

### 方案探索
- `openclaw backup create --only-config` → 仅 config.json（~1.5KB），但缺少 MEMORY.md 等
- `openclaw backup create --no-include-workspace` → `--dry-run` 显示仍包含 extensions/node_modules（~180MB）
- 结论：OpenClaw backup 命令过滤能力不足，需手动挑选

### 手动备份脚本
编写 `openclaw-config-backup.ps1`：
- 复制关键文件到临时目录
- tar 打包
- 推送到 GitHub

备份清单：
openclaw.json
config.json, config.yaml
identity/, devices/, cron/
memory/ + memory/self-improving/
workspace/memory/
workspace/{AGENTS,HEARTBEAT,MEMORY,TOOLS}.md

体积：~30KB ✅

### Git 历史清理
首次推送包含大文件对象，导致 `git push` 失败。
解决：删除 `.git`，重新 init，仅添加小备份文件， `git push --force` 成功。

### 回滚脚本
`restore-openclaw-config.ps1`：
- 自动备份当前状态（before-restore）
- 停止 gateway（尝试 scheduled task / service / process）
- 解压覆盖
- 重启 gateway

### PowerShell 别名
将 `boc`（备份）、`roc`（回滚）加入 `$profile`，永久生效。

### 验证流程
boc
修改配置文件
openclaw config validate
openclaw gateway restart
测试 → 异常则 roc

## 文件位置
- 备份脚本: `C:\Users\Administrator\openclaw-config-backup.ps1`
- 回滚脚本: `C:\Users\Administrator\restore-openclaw-config.ps1`
- 备份目录: `C:\Users\Administrator\openclaw-backups\`
- Git 仓库: `origin git@github.com:wang4869527/my-openclaw-backup.git`
